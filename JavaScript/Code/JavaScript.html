Commenting on JavaScript

When we write in this language, we aren’t using the same commenting techniques as HTML, JS uses two ways to comment: The asterisk and forward slash for multi line coding:

/* hello here is a comment

Using two forward slashes before text for in-line comments, which means to ignore the rest of the text on that line after the two forward slashes.

// Here is another comment

This is useful for other devs reading your code to understand what each chunk of your code is doing, and also useful for you in future referring back on something with a bit of context. I personally refer back alot so this will be helpful for that. Your welcome, future me.


Declaring Variables

This harks me back to Python, or what little I learnt of it on a phone app. Data is anything of value to the computer, and in JS, there are 8 total data types, which we’ll learn about as we go along:
underfined, null, boolean, string, symbol, bigint, number and object.

Computers can do mathematical equations with numbers, but not with strings, as strings are typed characters such as “Hello 123”. A variable can be used to store and minipulate a data type, by stating it without directly using it. It is not a mathematical variable as it merely creates a value for a set of data at different times, and can be written with letters, numbers, an underscore (“_”) or a $ sign. But it cannot begin with a number or include spaces. We can declare a variable in JS with the shorthand “var” preceding it and then state it’s custom label. For example:

var aName;

Note: The semi-colon ends the variable label.


Storing Variables with Assignment Operators

What is an assignment operator? Easy. It’s the equals sign (=). If we want to assign a variable to a data type such as a number, we declare the variable as learnt above, and then define the variable on the left side of the assignment operator, and it’s corresponding number value on the right of the assignment operator. From then on, the code will represent that variable with that specific number value:

var number5;
number5 = 5;

Sounds about right doesn’t it?



Assigning the Value of One Variable to Another Variable

If we want to create a variable with the same value of another variable, we simply use the assignment operator to say this variable is equal to that variable, and the value of the two is then the same. We define one (line 1), give it a value (line 2), then define the other (line 3) and give it the value of the first one (line 4):

var var1;
var1 = 1;
var var2;
var2 = var1;

See how that worked?



Initialise Variables with the Assignment Operator

We can cut the above code above down by declaring and defining a variable in one line of code rather than over two lines. Don’t forget the semi-colon at the end:

var var1 = 1;

Boom.


Uninitialised Variables

When a variable is declared and brought into the land of the living, it’s initial value is undefined. Writing code with an undefined variable will result in NaN or “Not a Number”. Strings with undefined variables will return the string as “undefined”. So, always remember to initialise your variables with a data type value!


Case Sensitivity in JavaScript

Variable labels like “variableOne” and “variableone” are not the same variable. Different variables can use the same words but in different casing. It’s strongly recommended that you practise writing variables in JS with camelCase, which uses the first word of a multi-word variable in lowercase and the second word in the variable label beginning with a capital letter.

Add or Subtract Two Numbers in JavaScript

We can do a bit of maths thanks to JS, and define numeric data with the data type Number. Using the addition operator, or the “+” symbol to all you earthly folks like me, we can add two numbers together in JS:

var myNum = 5 + 10;

Here, the variable myNum is declared and then assigned the value of 15. The same thinking can be achieved with subtraction in JS, using the “-” symbol.



Multiplication and Division in JavaScript

So we want to multiply two numbers together, we use the asterisk “*” between the two digits. Easy.

var myNum = 3 * 4;

The same goes for division. We use the forward slash “/” symbol to achieve a divide between two digits.

var myNum = 10 / 2;



Increment or Decrement a Number

We can easily add one or increment the number of a variable by using the ++ operator. Instead of writing:

i = i + 1;

We can shorten it to:

i++;

This can also be applied when we want to minus one or decrement the number of a variable by using the “- -” operator.

i--;

Note: In each case, the -- or ++ operator removes the need to add an equal sign.



Create Decimal Numbers

We can also declare a variable and assign it a decimal number, which are known as floating point numbers or floats.

var myDecimal = 5.7;



Multiplying or Dividing Decimals like Numbers

We can also multiply or divide decimal numbers as if they were numbers in JS. Simply put, we can multiply decimals as:

var myDecimal = 2.5 * 2.0;

or with division:

var myDecimal = 2.5 / 2.0;

Finding the Remainder in JS

The remainder operator (“%”) is used to check a number in JS as even or odd, when the remainder is checked against the number 2. Odd numbers will result in an answer of 1 whereas even numbers will give a null output:

48 % 2 = 0 (Even)

17 % 2 = 1(Odd)


Compound Assignment with Augmented Maths


Addition

When using assignments to modify the contents of a variable, remember that everything to the right of the equals sign is evaluated first, and if we want to do addition as well as an assignment, we can use the += operator to do both simultaneously. So if we say:

var myVar =1;

myVar += 5;

We are defining the variable “myVar” and to be equal to 1, then we add 5 to that variable, and perform the following to give the result which will equal 6:

console.log(myVar);


Subtraction

The same can be applied with subtraction operations by using -= instead of +=. For example:

var myVar = 1;

myVar -= 5;

console.log(myVar);

This will result in -4.


Multiplication

If we want to multiply and perform an assignment at the same time, we can use the *= operator. Same process of thought as before.


Division

Declaring String Variables

Escaping Quotations in Strings

Everything has a meaning in programming, even quotes within strings. They may be misread by JS, ah I miss HTML. So simple. var simpleString = “Then he looked, \“I have had a great day today\”, said John.”;

Note: It’s good practise to use double quotes ( “ ” )for a string and not single quotes ( ‘ ’ ) as you may need to use the single quotes in words like “it’s” for example. If you do so, always remember that backslash: “it\’s”.

Find the Length of a String

Cause why not ey.

Simply type .length after the string to see it’s length:

var myStr = “This is a string.”.length;

Note: Write .length immediately after the string literal or string variable.


Use Bracket Notation to Identify the First Letter of a String

Bracket Notation is one way to identify a character at a particular index (a point in the data) within a string. So you could also say [1] or [200] or whatever to identify wherever you choose to identify the Nth number of a string.

Remember how coding counts from 0 onwards, not 1. So the first letter in a word is the 0th letter. This is called zero-based indexing.

Let’s find out the first letter of the word “Charles”, by defining it as the variable “firstName” first, then find the first letter by defining the variable “firstLetter” using the Bracket Notation 0 for the variable “firstName”:

var firstName = “Charles”;

var firstLetter = firstName[0];

This will return the answer “C”.


Understanding String Immutability

One problem is that Strings cannot be amended (they are immutable) after they have been created. However, a string can be assigned a completely new value instead, this is a way to work around that immutability instead of trying to assign edits to the string. For example:

var myStr = “Hello”;

myStr[0] = “J”;

-> This will *not* change the string to “Jello”.

var myStr = “Hello”;

myStr = “Jello”;

-> This *will* change the string to “Jello”.


Find the Last Letter of a String

In the event that you want to find the last letter of a string, use this formula:

Within the Bracket Notation, minus 1 from the .length of the variable:

var myStr = “Charles”;

var lastLetter = myStr[myStr.length -1];

The answer will be “s”.

Note: Using a higher number will index letters in the Nth-to-last position. For example, the third last letter of “Charles” (the letter l) would be [myStr.length -3]. This means the code is counting back 3 places from the end point of the string. Understand me?


Storing Multiple Values in One Variable with JavaScript Arrays

It’s possible to assign multiple values to one variable by separating each one with a comma, all inside square brackets. This is called an array declaration. These can be strings, numbers…

Here’s an example:

var sandwich = [“cheese”, “ham”, “salad”];


Nesting Arrays

Remember what nesting was? That’s putting one thing inside another, and here we’re able to ‘nest’ an array inside another, like so:

[[“eggs”, “ham”], [“cheese”, “salad”]]

Note: This is also called a multi-dimensional array.


Access Array Data with Indexes

FreeCodeCamp likes to teach one one thing followed by another, and then mix them together. We have learn’t how to index with Bracket Notation, and how to create an Array, now we’re going to use a Bracket Notation on the variable myArray with three nested values, and index that data within myArray, for example:

var myArray = [10, 40, 100];

myArray[0];

This gives 10.

var myData = Array[1];

This gives 40.

Remember: To think about the zero-based indexing structure.


Modifying Array Data with Indexes

It’s worth noting here that entries of arrays are mutable, unlike strings which are immutable, and can be freely edited after creation. Let’s hark back to this string example earlier which would not work:

var myStr = “Hello”;

myStr[0] = “J”;

We can write virtually the same thing in the form of an array and it will work:

var myArray = [10, 20, 30];

myArray[0] = 15;

This will mean that the data inside myArray is now [15, 20, 30].
